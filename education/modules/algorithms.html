Quick Reference

Master Theorem (common case)

T(n) = a T(n/b) + f(n)
Let n^{log_b a} = n^k
- If f(n) = O(n^{k-ε}) → T(n) = Θ(n^k)
- If f(n) = Θ(n^k log^p n) → T(n) = Θ(n^k log^{p+1} n)
- If f(n) = Ω(n^{k+ε}) and regularity → T(n) = Θ(f(n))

Typical examples: mergesort (a=2,b=2,f=n) → Θ(n log n); Strassen-like recurrences vary k.

Greedy correctness (pattern)

1) Greedy choice property: show an optimal solution has a greedy-first choice.
2) Exchange argument: swap steps to transform any optimum to greedy form
   without lowering optimal value.

Huffman: merging two least-frequent symbols is always safe (exchange proof).

DP recipe

1) Define subproblem F(i, …)
2) Recurrence (optimal substructure)
3) Base cases
4) Order of evaluation (topo)
5) Transition complexity
6) Reconstruct solution (parents)

Memoise top-down first to validate, then convert to bottom-up for speed.

BFS / DFS complexity

Adjacency list:
BFS/DFS: Θ(V + E)
Topological sort: Θ(V + E)

Use adjacency lists for sparse graphs; matrices cost Θ(V²).

Code Snippets

Huffman coding (sketch)

# frequencies: dict{char: freq}
import heapq
heap = [[f, c] for c, f in frequencies.items()]
heapq.heapify(heap)
while len(heap) > 1:
  f1, n1 = heapq.heappop(heap)
  f2, n2 = heapq.heappop(heap)
  heapq.heappush(heap, [f1+f2, (n1, n2)])  # merge
# heap[0] holds the tree; traverse to assign codes

Time Θ(n log n) with n symbols (priority queue merges).

Dijkstra (array vs heap)

# Min-heap implementation: Θ((V+E) log V)
# Using array for 'extract-min': Θ(V²) on dense graphs
# Initialise dist[s]=0; relax edges via priority queue updates

Non-negative weights only; use Bellman-Ford for negatives.

Key Takeaways

  • Pick the right paradigm: greedy when exchange works, DP when subproblems overlap.
  • Prove before you optimise: correctness + complexity guide implementation choices.
  • Use the right data structures to hit the intended time bounds.
