Quick Reference

PySpark skeleton

from pyspark.sql import SparkSession, functions as F
spark = SparkSession.builder.appName("etl").getOrCreate()

src = spark.read.parquet("s3://bucket/raw/2025/*")
dim = spark.read.parquet("s3://bucket/dim/users/")

# Broadcast small dimension to avoid shuffle
dim_b = F.broadcast(dim)

facts = (
  src.join(dim_b, "user_id", "left")
     .withColumn("day", F.to_date("event_ts"))
     .groupBy("day", "country").agg(F.count("*").alias("events"))
)

facts.write.mode("overwrite").partitionBy("day").parquet("s3://bucket/curated/events_by_day")

Broadcast joins trade network shuffle for executor memory — ensure the dim table is small.

Partitions & caching

# Repartition large datasets by join key to reduce skew
df = df.repartition(200, "user_id")

# Cache hot intermediate results
wide = df.select("a","b","c","user_id").cache()
# ... multiple actions on 'wide' ...
wide.unpersist()

Use coalesce() to reduce partitions for small outputs; avoid over-partitioning tiny writes.

Windowed aggregations

from pyspark.sql.window import Window
w = Window.partitionBy("user_id").orderBy("ts").rowsBetween(-10, 0)
df = df.withColumn("rolling_10", F.sum("amount").over(w))

Prefer time-based windows for streaming; rowsBetween is batch-only semantics.

Structured Streaming (sketch)

stream = (spark.readStream.format("kafka")
  .option("subscribe","events").load())

parsed = stream.selectExpr("CAST(value AS STRING) json")  # parse JSON...

q = (parsed.writeStream
  .outputMode("append")
  .option("checkpointLocation","s3://bucket/chk/etl")
  .format("delta")  # or parquet
  .start("s3://bucket/bronze/events"))
q.awaitTermination()

Always set a durable checkpointLocation for exactly-once sinks.

Key Takeaways

  • Columnar formats + partitioning enable fast scans and pruning.
  • Most performance problems are shuffle or skew — fix joins first.
  • Design for retries and idempotency; storage is the system of record.
