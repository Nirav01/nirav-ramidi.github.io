Learning Outcomes

  • Plan and execute scaling & performance work (profiling, bottleneck removal, caching).
  • Achieve meaningful test coverage (unit/integration/e2e) and enforce via CI.
  • Set up CI/CD with build, test, lint, containerisation, and deploy gates.
  • Add observability (logs/metrics/traces) and define SLOs for the app.
  • Produce docs (README, runbooks, ADRs) and a clean handover.

Indicative Topics

Quality Engineering

Test pyramid, flaky tests triage, static analysis, mutation testing for critical paths.

Performance & Reliability

Profiling, caching, timeouts/retries/circuit breakers, graceful shutdown, health checks.

Delivery Pipelines

Build artifacts, container images, environment promotion, feature flags, rollback strategy.

Observability

Structured logs, metrics & alerts, tracing critical flows, error budgets & SLOs.

Documentation & Handover

ADRs, runbooks, API docs (OpenAPI), onboarding notes, release notes.

Quick Reference

Test Pyramid (at a glance)

  • Unit (many, fast) → logic correctness
  • Integration (some) → modules/services together
  • E2E (few, slow) → user-critical paths

Mock only where needed; prefer contract tests for interfaces.

GitHub Actions — basic CI

name: ci
on: { push: { branches: ["main"] }, pull_request: {} }
jobs:
  build_test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: "20" }
      - run: npm ci
      - run: npm test -- --ci --reporters=default
      - run: npm run lint

Extend with build artifacts, container image, and deploy jobs behind approvals.

Key Takeaways

  • Quality is engineered: tests + automation + observability make change safe.
  • Design for failure: timeouts, retries, and graceful degradation protect users.
  • Clear docs & handover reduce maintenance burden and onboarding time.
