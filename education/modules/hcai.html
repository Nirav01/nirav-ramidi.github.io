Human-Centred AI

Learning Outcomes

  • Apply explainability patterns (confidence, rationale, provenance, recourse) appropriate to the task.
  • Design and run user studies (within-subjects / A/B) with counter-balancing and clear metrics.
  • Measure trust, usability (SUS), and workload (NASA-TLX); report effect sizes and CIs.
  • Identify and mitigate fairness risks; communicate scope, limits, and guardrails.
  • Operationalise governance: model cards, decision logs, and feedback/appeal loops.

Indicative Topics

Principles

Appropriate use, calibrated transparency, contestability, privacy by design.

Design patterns

Confidence/uncertainty, progressive disclosure, counterfactuals/recourse, provenance/versioning.

Empirical methods

Task success/time, SUS, NASA-TLX, trust scales; paired t-tests/Wilcoxon, bootstrap CIs.

Fairness & ethics

Group metrics, harms/benefits framing, consent and data minimisation, documentation.

const log = (type, payload) => fetch("/log", {
  method: "POST",
  headers: {"Content-Type":"application/json"},
  body: JSON.stringify({
    ts: Date.now(), uid, session, type, ...payload
  })
});
// Example: decision taken
log("decision", {model_ver, input_hash, output, confidence});

Use pseudonymous IDs; avoid storing raw PII. Document fields in a data sheet.

Key Takeaways

  • Clarity and recourse often matter more than raw model accuracy for user outcomes.
  • Trust is calibrated by honest uncertainty and reversible actions, not persuasion.
  • Measure with purpose: pick metrics that reflect the decision context.
