Learning Outcomes

  • Explain the roles of each layer in the OSI/TCP-IP stack and how they interact.
  • Be familiar with the main topics in networking: link, network, transport, and application layers.
  • Gain practical experience analysing traffic and building simple client/server programs.

Indicative Topics

Layered Models

OSI vs TCP/IP; encapsulation/decapsulation; PDU names (frame, packet, segment).

Link & Physical

Ethernet (MAC, VLANs, ARP), Wi-Fi basics, error detection (CRC), duplexing & collisions.

Network Layer

IPv4/IPv6 addressing, subnetting/CIDR, ICMP, routing (static, RIP/OSPF/BGP), NAT.

Transport

UDP vs TCP, 3-way handshake/teardown, flow & congestion control, reliability.

Application

DNS, HTTP/1.1→2→3, TLS, email protocols; REST patterns and latency budgets.

Performance & Troubleshooting

BDP, latency components, packet loss, QoS; tools: ping, traceroute, tcpdump/Wireshark.

Quick Reference

OSI Layers

7  Application  (HTTP, DNS)
6  Presentation (TLS, encoding)
5  Session      (state, dialogs)
4  Transport    (TCP, UDP)
3  Network      (IP, ICMP, routing)
2  Data Link    (Ethernet, ARP, VLAN)
1  Physical     (bits, media)

TCP/IP model condenses 5–7 into 4 layers (App · Transport · Internet · Link).

TCP Handshake & Close

Connect:  SYN →  SYN+ACK →  ACK
Close:    FIN →  ACK  →  FIN →  ACK

Retransmissions/backoff handle loss; TIME_WAIT prevents old segments reappearing.

CIDR & Subnetting

Hosts per /n  = 2^(32−n) − 2   (for /n ≤ /30)
Example: 192.168.1.0/26 → 64 addrs → 62 hosts

/31 (point-to-point) and /32 (host route) are special cases without −2.

Latency & BDP

Total latency ≈ propagation + transmission + queueing + processing
BDP (bytes)    = bandwidth · RTT

Windows/buffers must cover the bandwidth-delay product for full throughput.

Lab Snippets

# Minimal TCP echo server (Python)
import socket
s = socket.socket()
s.bind(("0.0.0.0", 5000))
s.listen()
conn, _ = s.accept()
data = conn.recv(4096)
conn.sendall(data)
conn.close(); s.close()
# Inspect traffic quickly
ip addr            # addresses & interfaces
ip route           # routing table
tcpdump -i eth0 port 5000
wireshark          # GUI analysis

Good hygiene: warm up, capture both ends if possible, and annotate timestamps.
