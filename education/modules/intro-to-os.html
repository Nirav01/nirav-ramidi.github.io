Core OS concepts: processes & threads, CPU scheduling, concurrency & synchronisation, deadlocks, memory management & virtual memory, filesystems, and I/O with a taste of virtualisation.

Learning Outcomes

  • Explain OS abstractions (process, thread, address space, file, socket) and their lifecycle.
  • Compare scheduling strategies and reason about latency/throughput trade-offs.
  • Write correct synchronised code using mutexes, semaphores, and condition variables.
  • Analyse and avoid deadlocks via prevention/avoidance/detection strategies.
  • Describe paging, TLBs, and demand-paged virtual memory; reason about locality.
  • Understand filesystem layout (inodes, directories, journaling) and buffered I/O.

Indicative Topics

Processes & Threads

PCB/TCB, state transitions (new→ready→running→blocked→terminated), context switching, user vs kernel mode.

CPU Scheduling

FCFS, SJF/SRTF, RR, priority & multilevel queues; starvation & aging; metrics (waiting, turnaround, response).

Synchronisation

Critical sections, atomicity, locks, semaphores, monitors; producer–consumer & readers–writers patterns.

Deadlocks

Coffman conditions, resource-allocation graphs, Banker's algorithm, prevention/avoidance/detection & recovery.

Memory & VM

Paging & segmentation, page tables (multi-level), TLB, page replacement (LRU/Clock), thrashing & working sets.

Storage & Filesystems

Inodes, allocation strategies, journaling (ext4), caching & write-back, fsck basics.

I/O & Virtualisation

Device drivers, interrupts/DMA, polling vs interrupts, hypervisors & containers (namespaces/cgroups).

Quick Reference

Scheduling metrics

Waiting time   = Î£ (start_i − arrival_i)
Turnaround     = finish − arrival
Response time  = first_run − arrival

Shorter time-slices improve response but raise context-switch overhead.

Deadlock conditions

1. Mutual exclusion
2. Hold and wait
3. No preemption
4. Circular wait

Break any one to prevent deadlock (e.g., impose a global lock order).

Producer–Consumer (semaphores)

// Pseudo-code
semaphore empty = N, full = 0, mutex = 1
producer:
  wait(empty); wait(mutex);
  put(item); 
  signal(mutex); signal(full);

consumer:
  wait(full); wait(mutex);
  get(item);
  signal(mutex); signal(empty);

Address translation

virtual_addr = (VPN, offset)
phys_frame   = PT[VPN]  // TLB hit ≈ fast; miss → walk
physical_addr = (phys_frame, offset)

Locality (temporal/spatial) underpins caching & paging performance.

Key Takeaways

  • OS abstractions simplify programming by safely multiplexing hardware.
  • Concurrency demands disciplined synchronisation to avoid races & deadlocks.
  • Virtual memory enables isolation & overcommit; locality makes it fast in practice.
  • Filesystems balance durability and performance via journaling and caching.
