Learning Outcomes

  • Formulate business/engineering problems as mathematical programs (LP/ILP, flows, assignments).
  • Apply simplex and understand duality, shadow prices, and reduced costs.
  • Model integer constraints (binary decisions, logical constraints, big-M safely).
  • Use solvers via PuLP/JuMP; interpret solver output & sensitivity reports.
  • Design small heuristics (greedy, local search) when exact solves are impractical.

Indicative Topics

LP fundamentals

Standard form, geometry, feasible regions, basic feasible solutions, simplex pivoting.

Duality & sensitivity

Primal–dual pairs, complementary slackness, shadow prices, RHS/coeff ranges.

Integer programming

Branch-and-bound, cutting planes, modelling tricks (SOS, big-M, indicator constraints).

Network optimisation

Shortest paths, max-flow/min-cut, min-cost flow, assignment & transportation.

Heuristics

Greedy, local search, metaheuristic intuition for large combinatorial problems.

Quick Reference

LP — standard form

maximise   cᵀx
subject to Ax ≤ b
           x ≥ 0

Convert ≥ with slack/surplus & sign constraints; equality via two inequalities.

Dual (for ≤-primal)

minimise   bᵀy
subject to Aᵀy ≥ c
           y ≥ 0

Complementary slackness ties primal activity to dual feasibility.

Modelling Snippets

Python · PuLP (transport)

import pulp as pl

prob = pl.LpProblem("transport", pl.LpMinimize)
x = pl.LpVariable.dicts("x", (plants, markets), lowBound=0)

# Objective
prob += pl.lpSum(cost[i][j] * x[i][j] for i in plants for j in markets)

# Supply/ demand
for i in plants:  prob += pl.lpSum(x[i][j] for j in markets) <= supply[i]
for j in markets: prob += pl.lpSum(x[i][j] for i in plants) >= demand[j]

prob.solve()

Julia · JuMP (assignment)

using JuMP, GLPK
model = Model(GLPK.Optimizer)
@variable(model, x[1:n,1:n], Bin)
@objective(model, Min, sum(C[i,j]*x[i,j] for i=1:n, j=1:n))
@constraint(model, [i=1:n], sum(x[i,j] for j=1:n) == 1)
@constraint(model, [j=1:n], sum(x[i,j] for i=1:n) == 1)
optimize!(model)

Key Takeaways

  • Good formulations are often more powerful than clever heuristics.
  • Dual variables provide economic intuition (shadow prices) for constraints.
  • Be cautious with big-M; choose tight bounds to keep relaxations strong.
  • Network problems admit specialised fast algorithms—use them when applicable.
