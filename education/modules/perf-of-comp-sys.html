Learning Outcomes

    Lab Snippet — Simple Load Test

    # Using 'hey' (Go HTTP load generator)
hey -z 60s -q 0 -c 64 -m GET https://localhost:8080/api

# Using wrk
wrk -t8 -c128 -d60s --latency https://localhost:8080/api

    Warm up the system first; capture percentiles and CPU/memory/I/O metrics alongside RPS.

    Key Takeaways

    • Use models to predict and experiments to verify under real workloads.
    • Design to percentiles, not averages; tails drive user experience.
    • Keep utilisation modest at peak; small headroom prevents big tail spikes.
