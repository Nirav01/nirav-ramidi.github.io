Learning Outcomes

  • Apply the CIA triad and basic risk assessment to prioritise controls.
  • Use threat modeling (STRIDE) to discover threats and define mitigations.
  • Understand crypto building blocks (symmetric/asymmetric, hashes, MACs) and PKI.
  • Design authentication and authorisation (MFA, RBAC/ABAC, least privilege).
  • Harden web and network surfaces; log and respond to incidents.

Indicative Topics

Principles & Risk

CIA, AAA, least privilege, fail-safe defaults, risk registers, control selection.

Threat Modeling

STRIDE, data-flow diagrams, trust boundaries, mitigations & verification.

Crypto Basics

AES/GCM, RSA/ECDSA, hashes (SHA-2/3), KDFs (bcrypt/Argon2), TLS/PKI, key management.

Web & AppSec

OWASP Top 10, input validation/encoding, CSRF/XSS/IDOR, secure cookies, headers (CSP/HSTS).

Network & Ops

Network segmentation, firewalls/SGs, TLS everywhere, logging, SIEM, incident response runbooks.

Quick Reference

Password hashing (Python)

# pip install argon2-cffi
from argon2 import PasswordHasher
ph = PasswordHasher()
hash = ph.hash("correct horse battery staple")
ph.verify(hash, "correct horse battery staple")  # True

Use Argon2id or bcrypt; never store raw passwords.

OpenSSL — quick TLS check

# List server's cert chain & negotiated ciphersuite
openssl s_client -connect example.com:443 -servername example.com -showcerts

Pin to modern TLS; prefer AEAD ciphers (e.g., TLS 1.3 suites).

Security headers (Nginx)

add_header X-Frame-Options "DENY";
add_header X-Content-Type-Options "nosniff";
add_header Referrer-Policy "no-referrer";
add_header Content-Security-Policy "default-src 'self'";

Tune CSP for your assets; add HSTS after confirming HTTPS is stable.

STRIDE prompts

Spoofing      → strong auth, MFA
Tampering     → integrity checks, signatures
Repudiation   → audit logs, time-sync
Information   → access controls, encryption
Disclosure
Denial of Svc → rate limits, timeouts, WAF
Elevation     → least privilege, sandboxing

Key Takeaways

  • Security is a process: design, verify, monitor, and respond.
  • Use defence in depth: prevent, detect, contain, and recover.
  • Prefer secure defaults and least privilege everywhere.
