Learning Outcomes

  • Choose the right structure to meet time/space constraints.
  • Implement core structures with correct invariants and tests.
  • Analyse operations with precise asymptotics (worst/avg/amortised).
  • Apply structures inside algorithms (priority queues, maps/sets, balanced trees).

Quick Reference — Common Complexities

Operations

Dynamic array:
  index  O(1)     push_back  amort. O(1)
  insert/delete at i  O(n)
Linked list (singly):
  head ops O(1)   search O(n)

Hash table (avg):
  get/put  O(1)   worst O(n) if many collisions
BST (balanced):
  search/insert/delete  O(log n)
Heap (binary):
  push/pop-min  O(log n)   peek  O(1)

Stack via Deque (Java)

import java.util.ArrayDeque;
ArrayDeque st = new ArrayDeque<>();
st.push(10); st.push(20);
int top = st.pop();   // 20

Prefer ArrayDeque over legacy Stack.

Binary Search (Python)

def bsearch(a, x):
    lo, hi = 0, len(a)-1
    while lo <= hi:
        mid = (lo + hi)//2
        if a[mid] == x: return mid
        if a[mid] < x:  lo = mid + 1
        else:           hi = mid - 1
    return -1

Requires sorted array; runtime Î˜(log n).

Key Takeaways

  • Pick structures that match access patterns and mutation costs.
  • Preserve invariants (e.g., BST order, heap property) with every operation.
  • Use amortised analysis to reason about dynamic arrays and hash tables.
