Quick Reference

Java class & main()

public class Hello {
  public static void main(String[] args) {
    System.out.println("Hello, Java!");
  }
}

Compile with javac Hello.java; run with java Hello.

Exceptions

try {
  riskyOp();
} catch (IOException e) {
  // handle / log
} finally {
  cleanup();
}

Prefer specific exception types; avoid swallowing errors silently.

JUnit 5 test

import static org.junit.jupiter.api.Assertions.*;
import org.junit.jupiter.api.Test;

class MathUtilsTest {
  @Test void addsTwoNumbers() {
    assertEquals(5, MathUtils.add(2, 3));
  }
}

Arrange–Act–Assert; keep tests small and deterministic.

OOP vocabulary

  • Encapsulation: hide internals behind methods.
  • Inheritance: reuse/extend behaviour via extends.
  • Polymorphism: call through interface/supertype.
  • Interfaces: contracts; multiple per class.

Design Snippets

Class + interface

public interface Payable { double total(); }

public class LineItem implements Payable {
  private final String name;
  private final int qty;
  private final double price;
  public LineItem(String name, int qty, double price) {
    this.name = name; this.qty = qty; this.price = price;
  }
  @Override public double total() { return qty * price; }
}

Inheritance & overriding

abstract class Shape { abstract double area(); }
class Circle extends Shape {
  private final double r;
  Circle(double r){ this.r = r; }
  @Override double area(){ return Math.PI * r * r; }
}

Key Takeaways

  • Favour clear class design and small, testable methods.
  • Use interfaces to depend on contracts, not concrete classes.
  • Handle exceptions deliberately; fail fast with useful messages.
  • Automate tests early—regression safety boosts iteration speed.
