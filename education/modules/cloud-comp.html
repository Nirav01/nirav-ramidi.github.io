Learning Outcomes

  • Containerise apps and run them locally and in the cloud (Compose → basic K8s).
  • Pick appropriate compute/storage/networking services and understand trade-offs.
  • Apply IAM least-privilege, secrets management, and basic network security.
  • Automate infra with Terraform; ship via CI/CD pipelines.
  • Estimate and monitor costs; design for reliability with health checks and autoscaling.

Quick Reference

docker-compose.yml (dev)

version: "3.9"
services:
  api:
    build: .
    ports: ["8080:8080"]
    env_file: .env
    depends_on: [db]
  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: example
    volumes:
      - pgdata:/var/lib/postgresql/data
volumes:
  pgdata: {}

Use multi-stage Dockerfiles; keep images small and non-root.

GitHub Actions — build & push

name: ci
on: {push: {branches: ["main"]}}
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: docker/setup-buildx-action@v3
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/build-push-action@v6
        with:
          push: true
          tags: ghcr.io/<owner>/<repo>:sha-${{ github.sha }}

Pin base images; scan with Trivy/Grype in CI.

Key Takeaways

  • Design to the SLO: reliability/cost drive service choices more than hype.
  • IAM and network boundaries are your first line of defence—keep defaults tight.
  • Automation (IaC + CI/CD) reduces drift and speeds up safe iteration.
  • Prefer managed building blocks unless there’s a clear reason to self-manage.
