Learning Outcomes

  • Simplify logic with Boolean identities and Karnaugh maps.
  • Design adders, multiplexers, decoders, comparators, and small ALUs.
  • Use D/JK flip-flops to build registers, counters, and FSMs.
  • Specify and implement Mealy/Moore machines from textual requirements.
  • Analyse timing: setup/hold, clk→Q, critical path, and fmax.

Indicative Topics

Boolean Algebra & Minimisation

De Morgan’s, distributive/associative, absorption; K-maps (2–5 vars), don’t-care handling.

Combinational Blocks

Half/full adders, ripple/carry-lookahead, MUX/DEMUX, encoders/decoders, priority logic.

Sequential Logic

Latches vs flip-flops, registers, synchronous counters, shift registers, debouncing.

FSM Design

State encoding (binary/one-hot/Gray), transition/output logic, hazards & synchronisers.

Timing & Constraints

Setup/hold, clk→Q, propagation, skew/jitter, critical path and maximum clock frequency.

Quick Reference

De Morgan & Identities

¬(A · B) = ¬A + ¬B
¬(A + B) = ¬A · ¬B
A + (A · B) = A
A · (A + B) = A

Use don’t-cares (X) in K-maps to form larger groups (2ⁿ cells).

Adders — Ripple vs Lookahead

sum_i = a_i ⊕ b_i ⊕ c_i
c_{i+1} = g_i + p_i · c_i
g_i = a_i · b_i,   p_i = a_i ⊕ b_i

Carry lookahead reduces critical path depth vs ripple addition.

Flip-flops (characteristic)

D : Q_{next} = D
JK: Q_{next} = J·¬Q + ¬K·Q   // J=K=1 toggles

Use D-FFs for pipelines/registers; JK useful for toggle counters.

Clock & fmax

T_clk ≥ T_clk→Q + T_comb + T_setup + T_skew
f_max = 1 / T_clk

Critical path defines T_comb; shorten with pipeline registers.

Binary ↔ Hex Quick Map

0000=0 0001=1 0010=2 0011=3
0100=4 0101=5 0110=6 0111=7
1000=8 1001=9 1010=A 1011=B
1100=C 1101=D 1110=E 1111=F

FSM Steps

1) Define I/O & states
2) Draw state diagram
3) Write transition/output table
4) Choose encoding (one-hot/binary)
5) Derive next-state/output logic (K-maps)
6) Verify timing & reset
