Real-Time Switch Project

A high-performance real-time network switch implementation designed for low latency and high throughput. Built with C/C++, it handles packet forwarding, address learning, and switching logic with minimal overhead.

Problem & Goals

Modern networks require switches that can handle high traffic loads with minimal delay. The goal of this project was to design and implement a software-based switch that mimics the behavior of hardware switches, focusing on:

  • Real-time packet processing to ensure strict timing guarantees.
  • Efficient MAC address learning and lookup tables.
  • Handling concurrent traffic flows without blocking or dropping packets unnecessarily.

Architecture

The switch architecture is built around a central forwarding engine that interacts with multiple network interfaces. Key components include:

  • Ingress/Egress Queues: Buffers to handle bursty traffic.
  • Lookup Table: A hash map or similar structure for fast MAC-to-Port mapping.
  • Switching Logic: The core decision-making unit that determines where to forward packets.

Implementation Highlights

  • Optimized Data Structures: Used efficient hashing for O(1) average time lookups.
  • Zero-Copy Networking: Minimized data copying to reduce CPU usage and latency.
  • Concurrency Control: Implemented lock-free or fine-grained locking mechanisms to handle multi-threaded packet processing.

Challenges & Solutions

One of the main challenges was managing race conditions in the lookup table when multiple threads attempted to update or read from it simultaneously. This was resolved using atomic operations and careful synchronization.
