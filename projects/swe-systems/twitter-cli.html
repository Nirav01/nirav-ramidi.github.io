Twitter-like CLI (C)

A command-line microblogging system written in C: users can post, follow/unfollow, view a top-10 news feed, delete accounts, and advance time with endTurn. Final refactor replaces array storage with a linked-list news feed for simpler deletions and ordered traversal.

  • C (modular)
  • Linked-list feed
  • O(n) feed build
  • Unit & integration tests

Problem & Goals

Design a simple microblogging platform with a deterministic command set and a news feed that shows the 10 most recent tweets from a user and accounts they follow. Start with array-backed storage, then refactor the news-feed mechanism to a linked list to simplify insert/delete and maintain recency order.

  • Clear separation between I/O parsing and core logic.
  • Deterministic behaviour and safe memory management (no leaks/dangling pointers).
  • Graceful deletion of users and their tweets.

Data Model

Structs

  • Tweet: id (monotonic timestamp), text (≤280), username, *next
  • User: username, followers, following, tweets (head ptr)
  • TwitterSystem: dynamic arrays (users), global clock, temp buffers

Tweets are kept in per-user singly linked lists, newest at head.

Why linked list?

  • Delete user: drop list head and iterate freeing nodes (O(tweets)).
  • Post: push-front (O(1)) with current timestamp id.
  • Feed: k-way merge (users followed + self) by id; stop at 10 entries.

Command Interface

post

post <user> "<text up to 280 chars>" — creates a tweet with current clock as id.

follow / unfollow

follow <userA> <userB> — A follows B. unfollow reverses it (no-ops are safe).

newsfeed

newsfeed <user> — print up to 10 most recent tweets from user ∪ following, newest first.

delete

delete <user> — remove account and all tweets; detach from follower/following lists.

endTurn / endTwitter

endTurn increments the global clock; endTwitter flushes state and exits.

Implementation Highlights

  • Parsing separates tokens and quoted text; rejects invalid commands with helpful errors.
  • Memory safety: ownership is explicit; every malloc has a matching free on endTwitter / deletion paths.
  • Feed: k-way merge from heads of followed lists (and self); a small max-heap by id yields O(m log k) for m≤10.
  • Refactor: initial array solution replaced with linked lists to simplify deletions and head insertions.

Testing

  • Unit: follow/unfollow idempotence; delete user removes tweets; post length limits.
  • Integration: golden transcript comparison for a fixed command script.
  • Valgrind clean on all paths (no leaks, no invalid reads/writes).

What I learned

  • How data-structure choice (arrays → linked lists) changes complexity and simplifies edge cases.
  • Designing a stable CLI and separating I/O from logic improves testability.
  • Owning memory in C requires disciplined APIs and teardown paths.
