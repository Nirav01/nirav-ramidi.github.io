Healthcare Data Mining

Built end-to-end pipelines for an imbalanced stroke-risk dataset: cleaned features, applied SMOTE for minority up-sampling, compared classifiers, and explored clustering structure (KMeans/DBSCAN). Included a small regression track for numeric outcomes.

  • SMOTE (train-only)
  • ROC-AUC · PR-AUC · F1
  • KMeans · DBSCAN
  • Pipelines & CV

Dataset & Objectives

The primary task is binary classification of stroke vs no-stroke (high class imbalance). A secondary unsupervised task explores natural groupings that align with risk factors. An additional regression variant estimates a numeric health outcome (e.g., risk score).

  • Split: stratified train/test. Validation inside training via cross-validation.
  • Targets: binary label; optional numeric outcome for regression track.
  • Goal: high recall/PR-AUC on the positive class while keeping precision reasonable.

Preprocessing

Cleaning & Encoding

  • Handled invalid entries; imputed rare missing values with median/most-frequent.
  • One-hot encoded categoricals; ensured consistent feature order.
  • Standardised numeric features for distance-based/linear models.

Imbalance Handling

  • SMOTE applied inside the training fold only (to prevent leakage).
  • Compared to class-weighted learners as a baseline.

Pipelines & CV

  • scikit-learn Pipeline steps: preprocess → sampler → model.
  • Repeated/stratified CV; grid search with ROC-AUC (and PR-AUC) as scorers.
  • Threshold tuning on validation folds to target desired recall/precision.

Models

Classification

  • Logistic Regression (with class weights)
  • Random Forest / Gradient Boosting
  • SVM (RBF) with probability estimates
  • Baseline: majority class, dummy strategies

Clustering

  • KMeans (elbow + silhouette to select k)
  • DBSCAN (ε/MinPts tuned on scaled feature space)

Regression (light track)

  • Linear Regression / Ridge
  • RandomForestRegressor

Evaluation

Classification Metrics

  • ROC-AUC and PR-AUC (sensitive to imbalance)
  • Recall/Precision, F1 (macro & weighted)
  • Calibration check (reliability curve, Brier score)

Clustering Metrics

  • Silhouette score, Davies–Bouldin index
  • Qualitative inspection vs known risk factors

Regression Metrics

  • MAE, RMSE, and R² on held-out test set

Results

  • With SMOTE, tree-based models achieved meaningfully higher recall at similar ROC-AUC.
  • Threshold tuning (Youden’s J / PR-curve sweep) balanced precision vs recall for clinical use.
  • KMeans revealed clusters aligned with age/BMI/hypertension patterns; DBSCAN found small dense risk groups.
  • Regression baselines were stable after removing outliers and standardising features.

Lessons & Next Steps

  • Prevent leakage: apply SMOTE strictly within CV folds; keep test untouched.
  • Prefer PR-AUC for highly imbalanced problems; use calibration before decision thresholds.
  • Next: add XGBoost/LightGBM, cost-sensitive learning, SHAP explanations, and drift monitoring.
