Cloud-Native Library Analytics

Built a containerised ETL/ELT stack for library operations: ingested circulation & web events, modelled a star schema, shipped dashboards, and wired alerts for data freshness and anomalies.

  • Docker Compose
  • Airflow + dbt
  • Postgres / BigQuery
  • Metabase / Grafana
  • Freshness SLO < 2h

Problem & Goals

Library teams needed unified reporting across checkouts/returns/renewals, patron activity, and website usage. Manual CSV spreadsheets were inconsistent and stale. The goal: a reproducible pipeline and self-serve dashboards with a data freshness SLO < 2 hours.

  • Automate ingestion from ILS exports (CSV/SFTP), web logs, and branch spreadsheets.
  • Conform data to a clean warehouse model and document definitions.
  • Expose KPIs with filters (branch, category, time) and set up alerts for anomalies/freshness.

Architecture

Ingestion

  • Airflow DAGs pull daily ILS CSVs (SFTP) and parse Nginx access logs.
  • Idempotent loads with file hashing; partitioned staging tables by date.

Modelling

  • dbt models: staging → core → marts with tests (unique/not null, referential).
  • Star schema facts: fact_circulation, fact_visits; dims: dim_item, dim_branch, dim_time, dim_patron.

Serving

  • Metabase dashboards for circulation trends, popular titles, overdue rates, and visit heatmaps.
  • Grafana panels for pipeline health (task durations, success rate) and freshness SLIs.

Monitoring

  • Prometheus scrapes Airflow/exporters; alerts on freshness > 2h, task failures, row-count anomalies.
  • Great Expectations (optional) for data quality gates on critical tables.

Features

Unified KPIs

Checkouts/returns, overdue rate, visits, and conversion from catalog searches to checkouts.

Branch & Category Filters

Slice by branch, material type (books/media), and patron segments.

Freshness & Anomalies

Alerts when tables lag or when daily volumes deviate > 3Ïƒ from rolling mean.

Governance

Data dictionary and model lineage; versioned transformations with dbt.

Results

  • Replaced manual Excel workflows; recurring monthly report builds in minutes instead of hours.
  • Achieved < 2h freshness for core marts with reliable DAG retries and backfills.
  • Surfaced actionable signals (overdue spikes by branch/time; search terms with low checkout conversion).

Lessons & Next Steps

  • Clear dimensions (branch, item, patron) reduce dashboard confusion more than fancy charts.
  • Freshness SLIs + alerts create accountability and trust in dashboards.
  • Next: incremental materialisations, column-level lineage, and a small streaming path (Kafka) for near-real-time counts.
