Event Streaming Platform

Java · Kafka · Spring · PostgreSQL

Summary

Real-time ingestion and aggregation for analytics; reduced latency to 45 seconds at peak 120k events/min.

Context

  • Problem: Batch ETL created 15-minute stale dashboards.
  • Constraints: Commodity hardware, strict cost cap, near-real-time.

Role & Stack

Role: Sole backend engineer (design → delivery).

Stack: Java 21, Spring Boot 3, Kafka, Schema Registry, Debezium, PostgreSQL, Redis.

Architecture

  • Exactly-once semantics via idempotent producers and transactional writes.
  • Schema evolution with Avro + SR; dead-letter topic for invalid events.

Key Challenges & Solutions

Backpressure & burst handling

Implemented bounded queues + rate-aware consumers; autoscaled partitions after throughput tests.

Stateful aggregations

Windowed aggregations persisted to RocksDB; periodic snapshots to Postgres for BI.

Results

  • Latency: 15 min → 45 s.
  • Cost: within $420/mo infra budget.
  • Uptime: 99.95% over last quarter.
